# Backlog

This directory stores active implementation backlogs for work that is planned but not yet completed.

Rules:
- Create one file per active initiative using `YYMMDD-{backlog-title}.md`.
- Keep each file focused on one deliverable or tightly-related group of deliverables.
- Include context, scope, proposed approach, open questions, and clear exit criteria.
- Update the file while the work is active so it stays useful as the current source of truth.
- Delete the file once the work is fully implemented and no pending action remains.

Recommended sections:
- `Context`
- `Scope`
- `Out of Scope`
- `Implementation Notes`
- `Open Questions`
- `Exit Criteria`
